# Security and privacy reports

Please do not publish an exploit, privacy leak or dependency credential in a public issue.

Use GitHub’s private vulnerability reporting for this repository. Include the affected version, a minimal reproduction, impact and any safe mitigation you already found. Reports involving uploaded-image leakage, persistence without consent, unsafe project-file parsing or export injection are treated as security issues.

The project has no server-side user database, account system or runtime API key. That narrows the attack surface; it does not abolish it.
