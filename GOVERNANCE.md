# Governance

Letters & Light is maintainer-led and evidence-open.

pitch.dog sets product direction and release boundaries. Contributors can challenge any decision with a reproducible case, a clearer model or stronger user evidence. Authority is not immunity from being wrong.

Small reversible changes can merge after review and green checks. Changes to recommendation behavior, privacy, licenses, saved-project compatibility, accessibility contracts or public deployment need explicit maintainer review. Releases are tagged from a checked commit; a passing build alone is not a release.
